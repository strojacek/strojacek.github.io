> Going from Dap, to Dap2.


Previously I have worked on Gnu Dap, mainly adding some features to convert the files to .sas (instead of .sbs), and to add some patches to get the system to work on Termux. Though I enjoyed the process, I kept running into roadblocks. Mainly because if a .sas file I had to work on using Gnu Dap had do loops, or cards/datalines, or macros, the system wouldn't run.

This was a frustraing feeling, mainly as SAS (while arguably playing 2nd fiddle to R in my area of focus), is very heavily used in my field of study. SAS is also very expensive, and having an open source option that can provide some of the functionality will still be useful to quite a few people. So I decided, why not solve the problem by taking a very simple approach to implementing these features from scratch, and documenting it along the way?

So as an example, let's take the following code:

```sas
data test;
input x y z;
cards;
1 2 3
4 5 6
7 . 9
;
run;

proc print;
run;
```
