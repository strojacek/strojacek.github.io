Introduction

The purpose of this series is to teach statistics and machine learning, by designing a programming language that is based on SAS. 
In each post, I will implement a feature of the language, and explain the principles behind it.

Preliminaries
---

There are two things to consider when making a programming language.

1. What programming language do I write my new language in?
2. How do I parse my language?


I am implementing the lexer and parser of the language in ANTLR (I will explain lexing, parsing and ANTLR shortly), and starting the base implementation of Dap2 in Python (which you can find [here](https://github.com/strojacek/dap2)).


A brief overview of ANTLR
---

From a very high level, ANTLR is a tool designed to parse text. 
In order to do this, ANTLR breaks down the problem into three steps:

1. Grammar definition: You define the structure of the input using a grammar. This is like specifying the rules for how data or queries should be formatted.

2. Parsing: ANTLR helps convert structured text into a tree-like structure that reflects the hierarchical organization of the input. As an example, if you are using ANTLR to parse SQL statements, you can separate the logic for parsing the creation of a new table, and filtering data in an existing table. 
3. Code Generation: ANTLR can automatically generate code in various programming languages (like Java, C#, Python) that can then be used to process or analyze the input data according to the grammar you’ve defined.


In essence, ANTLR helps in creating tools that can understand and manipulate complex textual data, making it easier to handle and analyze the text according to the programmer's specification.
